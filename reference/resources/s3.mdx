## Overview

Amazon S3 (Simple Storage Service) provides scalable object storage for files, backups, static assets, and media. While Convox doesn't provision S3 buckets automatically, you can easily connect your application to S3 buckets using resource overlays.

## Setup Method

S3 buckets are connected to Convox applications via resource overlays - manually setting environment variables that contain S3 credentials and configuration.

## Creating an S3 Bucket

Create an S3 bucket using the AWS CLI, Console, or Infrastructure as Code:

<Tabs>
  <Tab title="AWS CLI">
    ```bash
    # Create bucket
    aws s3 mb s3://my-app-uploads --region us-east-1
    
    # Enable versioning (recommended)
    aws s3api put-bucket-versioning \
      --bucket my-app-uploads \
      --versioning-configuration Status=Enabled
    
    # Enable encryption (recommended)
    aws s3api put-bucket-encryption \
      --bucket my-app-uploads \
      --server-side-encryption-configuration '{
        "Rules": [{
          "ApplyServerSideEncryptionByDefault": {
            "SSEAlgorithm": "AES256"
          }
        }]
      }'
    ```
  </Tab>
  <Tab title="Terraform">
    ```hcl
    resource "aws_s3_bucket" "uploads" {
      bucket = "my-app-uploads"
    }
    
    resource "aws_s3_bucket_versioning" "uploads" {
      bucket = aws_s3_bucket.uploads.id
      versioning_configuration {
        status = "Enabled"
      }
    }
    
    resource "aws_s3_bucket_server_side_encryption_configuration" "uploads" {
      bucket = aws_s3_bucket.uploads.id
      rule {
        apply_server_side_encryption_by_default {
          sse_algorithm = "AES256"
        }
      }
    }
    ```
  </Tab>
</Tabs>

## IAM Permissions

Create an IAM user or role with S3 access:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:PutObject",
        "s3:GetObject",
        "s3:DeleteObject",
        "s3:ListBucket"
      ],
      "Resource": [
        "arn:aws:s3:::my-app-uploads",
        "arn:aws:s3:::my-app-uploads/*"
      ]
    }
  ]
}
```

<Note>
For production, use IAM roles when running on AWS (via IRSA for EKS) instead of static credentials.
</Note>

## Connecting to Convox

Define a placeholder resource in `convox.yml`:

```yaml
resources:
  uploads:
    type: s3

services:
  web:
    resources:
      - uploads
```

<Warning>
The `s3` resource type is not natively supported by Convox. This is a placeholder that you'll override with environment variables.
</Warning>

Set the environment variables using resource overlays:

```bash
convox env set \
  UPLOADS_BUCKET=my-app-uploads \
  UPLOADS_REGION=us-east-1 \
  UPLOADS_ACCESS_KEY_ID=AKIAIOSFODNN7EXAMPLE \
  UPLOADS_SECRET_ACCESS_KEY=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY \
  -a myapp -r production
```

## Environment Variables

Your application will have access to these environment variables:

<ParamField path="RESOURCE_BUCKET" type="string" required>
  The S3 bucket name
</ParamField>

<ParamField path="RESOURCE_REGION" type="string" required>
  The AWS region where the bucket is located
</ParamField>

<ParamField path="RESOURCE_ACCESS_KEY_ID" type="string">
  AWS access key ID (not needed if using IAM roles)
</ParamField>

<ParamField path="RESOURCE_SECRET_ACCESS_KEY" type="string">
  AWS secret access key (not needed if using IAM roles)
</ParamField>

## Usage Examples

<Tabs>
  <Tab title="Node.js">
    ```javascript
    const AWS = require('aws-sdk');
    
    const s3 = new AWS.S3({
      accessKeyId: process.env.UPLOADS_ACCESS_KEY_ID,
      secretAccessKey: process.env.UPLOADS_SECRET_ACCESS_KEY,
      region: process.env.UPLOADS_REGION
    });
    
    // Upload a file
    async function uploadFile(fileBuffer, key) {
      const params = {
        Bucket: process.env.UPLOADS_BUCKET,
        Key: key,
        Body: fileBuffer,
        ContentType: 'image/jpeg'
      };
      
      return await s3.upload(params).promise();
    }
    
    // Get a file
    async function getFile(key) {
      const params = {
        Bucket: process.env.UPLOADS_BUCKET,
        Key: key
      };
      
      return await s3.getObject(params).promise();
    }
    
    // Generate presigned URL
    function getPresignedUrl(key, expiresIn = 3600) {
      return s3.getSignedUrl('getObject', {
        Bucket: process.env.UPLOADS_BUCKET,
        Key: key,
        Expires: expiresIn
      });
    }
    ```

## Security Considerations

### Bucket Policies

Restrict public access:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Deny",
      "Principal": "*",
      "Action": "s3:*",
      "Resource": [
        "arn:aws:s3:::my-app-uploads",
        "arn:aws:s3:::my-app-uploads/*"
      ],
      "Condition": {
        "Bool": {
          "aws:SecureTransport": "false"
        }
      }
    }
  ]
}
```

This policy enforces HTTPS for all requests.

### Block Public Access

Enable block public access settings:

```bash
aws s3api put-public-access-block \
  --bucket my-app-uploads \
  --public-access-block-configuration \
    "BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true"
```

<Warning>
Never commit AWS credentials to version control. Always use environment variables or IAM roles.
</Warning>
