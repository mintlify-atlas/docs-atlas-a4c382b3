The `env` command group provides tools for managing environment variables in your Convox applications. Environment variables are key-value pairs used to configure your application at runtime.

## env

List all environment variables for an application.

### Usage

```bash
convox env
```

### Flags

<ParamField path="--app" type="string" default="-a">
  Application name
</ParamField>

<ParamField path="--rack" type="string" default="-r">
  Rack name
</ParamField>

<ParamField path="--release" type="string">
  Show environment variables from a specific release ID
</ParamField>

### Example Output

```bash
$ convox env
COUNT=0
FOO=bar
TEST=dummy
DATABASE_URL=postgres://...
```

## env edit

Edit environment variables interactively using your default editor.

### Usage

```bash
convox env edit
```

### Flags

<ParamField path="--app" type="string" default="-a">
  Application name
</ParamField>

<ParamField path="--rack" type="string" default="-r">
  Rack name
</ParamField>

<ParamField path="--promote" type="boolean" default="-p">
  Automatically promote the release after setting variables
</ParamField>

<ParamField path="--release" type="string">
  Edit environment from a specific release ID
</ParamField>

### Example

```bash
$ convox env edit
Setting FOO, BAR, DATABASE_URL... OK
Release: RABCDEFGHI
```

<Note>
  The editor used is determined by the `EDITOR` environment variable. If not set, it defaults to `vi`.
</Note>

### Interactive Editing

When you run `convox env edit`, it:

1. Opens a temporary file with your current environment variables
2. Launches your editor
3. When you save and close, uploads the new environment
4. Creates a new release with the updated variables

## env set

Set one or more environment variables.

### Usage

```bash
convox env set <key=value> [key=value]...
```

### Arguments

<ParamField path="key=value" type="string" required>
  One or more environment variable assignments
</ParamField>

### Flags

<ParamField path="--app" type="string" default="-a">
  Application name
</ParamField>

<ParamField path="--rack" type="string" default="-r">
  Rack name
</ParamField>

<ParamField path="--id" type="boolean">
  Output only the release ID
</ParamField>

<ParamField path="--promote" type="boolean" default="-p">
  Automatically promote the release after setting variables
</ParamField>

<ParamField path="--release" type="string">
  Base the new environment on a specific release ID
</ParamField>

<ParamField path="--replace" type="boolean">
  Replace all environment variables (removes existing ones not specified)
</ParamField>

### Examples

#### Set a Single Variable

```bash
$ convox env set FOO=bar
Setting FOO... OK
Release: RABCDEFGHI
```

#### Set Multiple Variables

```bash
$ convox env set FOO=bar BAZ=qux API_KEY=secret
Setting FOO, BAZ, API_KEY... OK
Release: RABCDEFGHI
```

#### Set from File

You can pipe environment variables from a file:

```bash
$ cat .env | convox env set
Setting DATABASE_URL, REDIS_URL, API_KEY... OK
Release: RABCDEFGHI
```

#### Set and Promote

Automatically promote the new release:

```bash
$ convox env set FOO=bar --promote
Setting FOO... OK
Release: RABCDEFGHI
Promoting RABCDEFGHI...
OK
```

#### Replace All Variables

Replace all environment variables with a new set:

```bash
$ convox env set FOO=bar BAZ=qux --replace
Setting FOO, BAZ... OK
Release: RABCDEFGHI
```

<Note>
  When using `--replace`, any environment variables not specified will be removed. Use with caution.
</Note>

## Understanding Releases

When you change environment variables, Convox creates a new **release**. A release is immutable and contains:

- Build ID
- Environment variables
- Timestamp

To apply environment changes to running processes, you must **promote** the release:

```bash
# Set variable (creates release)
convox env set FOO=bar
# Output: Release: RABCDEFGHI

# Promote the release
convox releases promote RABCDEFGHI

# Or use --promote flag to do both
convox env set FOO=bar --promote
```

## Best Practices

### 1. Never Commit Secrets

Store sensitive values as environment variables, not in code:

```bash
# Good
convox env set API_KEY=secret_value

# Bad - don't put secrets in code
# const API_KEY = "secret_value";
```

### 2. Use Descriptive Names

Use clear, consistent naming:

```bash
convox env set DATABASE_URL=postgres://...
convox env set REDIS_URL=redis://...
convox env set STRIPE_API_KEY=sk_live_...
```

### 3. Document Required Variables

Keep a `.env.sample` file in your repository:

```bash
# .env.sample
DATABASE_URL=postgres://user:pass@host:5432/db
REDIS_URL=redis://localhost:6379
API_KEY=your_api_key_here
```

### 4. Environment-Specific Variables

Use different values for staging and production:

```bash
# Staging
convox env set NODE_ENV=staging --app myapp-staging

# Production
convox env set NODE_ENV=production --app myapp
```

## Common Workflows

### Copy Environment Between Apps

```bash
# Export from one app
convox env --app myapp > env.txt

# Import to another app
cat env.txt | convox env set --app myapp-staging
```

### Backup Environment Variables

```bash
convox env > env-backup-$(date +%Y%m%d).txt
```

### Promote After Multiple Changes

```bash
# Make multiple changes
convox env set FOO=bar
convox env set BAZ=qux
convox env unset OLD_VAR

# Get latest release ID
RELEASE=$(convox releases -l 1 | tail -1 | awk '{print $1}')

# Promote once
convox releases promote $RELEASE
```
