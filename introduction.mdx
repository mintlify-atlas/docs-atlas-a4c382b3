# Welcome to Convox

Convox is an open-source Platform as a Service (PaaS) that runs on your own cloud infrastructure. Built on Kubernetes, Convox simplifies application deployment and management across multiple cloud providers while giving you complete control over your infrastructure.

## What You Can Do With Convox

Convox provides everything you need to run production applications:

- **Deploy Applications**: Simple `convox deploy` command handles building, pushing, and deploying your code
- **Manage Infrastructure**: Postgres, Redis, MySQL databases and other resources defined in your manifest
- **Scale Services**: Horizontal autoscaling based on CPU/memory metrics or static process counts
- **Monitor & Debug**: Stream logs, execute one-off commands, and inspect running processes
- **Rollback Instantly**: Revert to any previous release in seconds with zero downtime
- **CI/CD Integration**: Works seamlessly with GitHub Actions, GitLab CI, CircleCI, and other platforms

## Core Concepts

### Rack

A **Rack** is your Convox platform installation - an isolated set of computing resources, networking, and storage running on your cloud provider. Think of it as your personal PaaS.

<CodeGroup>
```bash List Racks
$ convox racks
NAME               PROVIDER  STATUS
org/production     aws       running
org/staging        aws       running
```

```bash Rack Info
$ convox rack -r production
Name      production
Provider  aws
Router    router.0a1b2c3d4e5f.convox.cloud
Status    running
Version   3.0.0
```
</CodeGroup>

Most teams use multiple Racks to separate environments:
- **Development**: Local Rack on developer workstations
- **Staging**: Cloud Rack for testing
- **Production**: Cloud Rack for live applications

### App

An **App** is a logical container for your application code and its infrastructure dependencies. Apps are defined by a `convox.yml` manifest and deployed as atomic units.

```yaml convox.yml
services:
  web:
    build: .
    port: 3000
    health: /health
resources:
  database:
    type: postgres
```

### Services

**Services** are the horizontally-scalable components of your application. Each service runs one or more processes and can expose network ports.

```bash
$ convox services
SERVICE  DOMAIN                               PORTS
web      web.myapp.0a1b2c3d4e5f.convox.cloud  443:3000
api      api.myapp.0a1b2c3d4e5f.convox.cloud  443:8080
```

### Resources

**Resources** are managed infrastructure components like databases, caches, and queues that your services depend on.

<Info>
Convox automatically provisions, configures, and injects connection credentials as environment variables into your services.
</Info>
